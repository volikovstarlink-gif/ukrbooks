# Рабочий режим: автопилот для жены (non-technical user)

**ЖЕЛЕЗНОЕ ПРАВИЛО: СНАЧАЛА ЛОКАЛЬНО. Никаких `git push` / деплоев в прод до того, как она сказала слово «прод» (или явно одобрила деплой).**

Порядок работы в каждой сессии с женой:

1. **Правки только локально.** Редактируй файлы, никаких git-команд на этом шаге. Не спрашивай разрешения на обычные правки — работай в автопилоте.
2. **Запусти dev-сервер, если не запущен:** `npm run dev` в фоне. Если уже работает — просто жди HMR.
3. **Активно открой браузер с её изменением.** Не пиши «посмотри на localhost» — сам открой нужную вкладку через `start http://localhost:3000/<путь-к-странице>` (Windows) так, чтобы она увидела именно тот экран, где видно правку. Если правка на конкретной странице — открывай эту страницу, не главную.
4. **Спроси по-русски:** "Нравится? Когда скажешь «прод» — зальём."
5. **Деплой в прод только когда она скажет «прод»** (или явно одобрит). Тогда: `git add` + `git commit` + `git push origin main`. Vercel сам задеплоит на ukrbooks.ink.
6. **Если «нет» / «поменяй» / что-то кроме слова «прод»** → продолжай итерации локально, каждый раз открывая свежую вкладку браузера с результатом. НЕ трогай git. НЕ деплой.

Общайся с ней по-русски. Не объясняй технические детали, если она не просит. Прод = `git push` в `main` (Vercel подхватывает автоматически) — относись к нему с соответствующей осторожностью.
